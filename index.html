Haupia ("how-PEE-ah")

A component library of tasty little blocks, made with a few simple ingredients.

No build step. No JavaScript. No CSS processing. Just HTML and CSS, with simple APIs.

by Jeff Yaus

Overview

Haupia is a lightweight (just 31K minified!) component library for webpages, designed to be simple to use, to install, and to modify. It is made from just pure CSS, meaning no build steps, no JavaScript dependencies, no CSS processing to configure, no image directory to set up. While maybe not as feature-rich as some component libraries, Haupia is ideal when you want a minimum of fuss. Just drop the stylesheet onto your page, and you're off!

(Haupia is named for the jelly-like Hawaiian dessert made from coconut milk, cornstarch, sugar, and salt — a treat that is cut into little blocks. "Tasty little blocks made from a few simple ingredients" embodied the spirit of this component library.)

Usage

Download it! Download it minified Fork it on GitHub

To install Haupia, just copy the stylesheet into your site, and link to it from your page. That's it! Go (coco)nuts!

Haupia only styles the CSS classes that are part of its API; it won't affect anything else on your page, so it is easily used alongside your existing CSS, or even alongside another front-end framework.

This simpler components just use the relevant HTML tag, plus a class of haupia- and the name of the element. For instance, a Haupia checkbox is a regular <input type="checkbox"> plus the class haupia-checkbox. More complex components might need additional wrapper or sibling elements. (Code examples are given below.)

Themes

The default theme is built around a bunch of CSS variables, found at the top of the stylesheet. These are easily modified to fit your site's own palette, typography, and look-and-feel. For instance, you can change the background color of the basic buttons by adjusting the value of --haupia-button-background-color to your preferred color.

Progress

This is a styled version of the <progress> tag, which shows the completion of a task. (To show progress along a series of steps, see Progress List below. To show a fractional value, see Meter below.)

<progress class="haupia-progress" value="70" max="100"></progress>

Progress List

The progress list shows the current progress along a series of labeled steps.

  • Completed step
  • Current step
  • Future step
<ul class="haupia-progress-list"> <li class="haupia-progress-list-complete">Completed step</li> <li class="haupia-progress-list-current" aria-current="step">Current step</li> <li>Future step</li> </ul>

Meter

This is a styled version of the <meter> tag, which shows a fractional value. (To show the amount of progress along a task, see Progress above.)

If we establish thresholds for a "low" value at 33%, "high" at 66%, and "optimum" at 90%:





<meter class="haupia-meter" min="0" max="100" low="33" high="66" optimum="90" value="75"></meter>

Browser Support

What makes Haupia's code so simple is its extensive use of CSS custom properties (aka CSS variables) and the :has selector, so it works in any browser that supports these features (which is basically all modern browsers).

Accessibility

Haupia's listed APIs include as much accessibility support as HTML and CSS alone can provide. But Haupia is a CSS-only library, and full accessibility support for your page may sometimes require JavaScript. Please follow the WCAG guidelines.

Haupia provides a visible focus-visible indicator for its components. If you want this style to apply to all focusable elements on your page, so that your page is visually consistent, add this CSS to your page: 

            
*:focus {outline: var(--haupia-focus-outline);}

*:focus-visible p {
  outline: var(--haupia-focus-visible-outline); 
  outline-offset: var(--haupia-focus-visible-outline-offset)
}
